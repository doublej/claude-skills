# Banned Words and Phrases

If any of these appear in a draft, replace with plain language or delete entirely.

## AI Slop Words

These words signal AI-generated text. Never use them:

- dive / deep-dive
- delve
- unlock
- unleash
- harness
- leverage (as verb)
- robust
- comprehensive
- holistic
- synergy / synergize
- pivotal
- transformative
- cutting-edge
- groundbreaking
- revolutionize
- streamline
- empower
- elevate
- foster
- navigate (metaphorical)
- landscape (metaphorical)
- ecosystem (unless actual biology)
- paradigm
- seamless / seamlessly
- supercharge
- spearhead
- bolster
- underpin
- multifaceted
- nuanced (as filler)

## AI Slop Phrases

These phrases are dead giveaways. Kill on sight:

- "In today's fast-paced world"
- "In today's rapidly evolving landscape"
- "Game-changer" / "a real game-changer"
- "Deep dive into"
- "Let's explore"
- "Let's delve into"
- "Excited to announce"
- "I'm thrilled to share"
- "Without further ado"
- "At the end of the day"
- "It goes without saying"
- "Take it to the next level"
- "Harness the power of"
- "Unlock the potential of"
- "In the realm of"
- "It's worth noting that"
- "Needless to say"
- "The key takeaway is"
- "Moving forward"
- "Best-in-class"
- "World-class"
- "One-stop shop"
- "Low-hanging fruit"
- "Think outside the box"
- "Silver bullet"

## Clickbait Title Patterns

Never use these title structures:

- "X Things You Need to Know About Y"
- "The Ultimate Guide to X"
- "You Won't Believe What Happened When X"
- "Why X Is the Future of Y"
- "X Is Dead. Long Live Y."
- "How X Is Revolutionizing Y"
- "Everything You Need to Know About X"
- "The Complete Guide to X"
- "X, Pair it With Y for Z"
- "Stop Doing X. Start Doing Y."
- "I Tried X for 30 Days. Here's What Happened."

### What to Do Instead

Write titles that honestly describe the content:

| Bad | Good |
|-----|------|
| "The Ultimate Guide to Caching" | "Caching strategies that actually work in production" |
| "You Won't Believe This Redis Trick" | "Using Redis pub/sub for real-time cache invalidation" |
| "10 Things Every Developer Needs to Know" | "Common mistakes in error handling (and what to do instead)" |
